# Music Player v1 手工测试清单

## 测试范围

- 壳层页面四主区：`LibraryPage`、`PlaylistPage`、`NowPlayingPage`、`SettingsPage`
- 播放核心流程：加载文件、播放控制、进度与音量
- 快捷键固定预设：`Ctrl+P`、`Ctrl+Left`、`Ctrl+Right`、`Ctrl+F`

## 一、核心流程

- [ ] 启动应用后，确认四主区均可见且布局未塌陷，底部播放栏可见。
- [ ] 在曲库输入有效目录，点击“扫描”，确认曲目数量更新且列表展示正常。
- [ ] 在曲库搜索框输入关键字，确认列表可实时过滤。
- [ ] 在播放栏点击“播放/暂停”，确认状态文案与按钮文案随状态切换。
- [ ] 点击“上一首/下一首”，确认曲目切换且标题、进度、状态同步变化。
- [ ] 拖动进度条与音量滑块，确认播放位置和音量变化符合预期。

## 二、异常流程

- [ ] 曲库目录为空字符串时点击“扫描”，确认应用无崩溃、无异常弹窗。
- [ ] 选择不可播放文件或无效文件路径，确认错误信息可见且应用保持可操作。
- [ ] 播放列表仅 1 首时触发上一首/下一首，确认无崩溃、按钮状态与逻辑一致。
- [ ] 在未加载任何歌曲时触发播放相关操作，确认应用稳定且状态提示合理。

## 三、快捷键验证

### 1) 播放控制快捷键

- [ ] 按 `Ctrl+P`，确认触发播放/暂停（等价于 `appController.playbackService.togglePlayPause()`）。
- [ ] 按 `Ctrl+Left`，确认切到上一首（等价于 `appController.playbackService.previous()`）。
- [ ] 按 `Ctrl+Right`，确认切到下一首（等价于 `appController.playbackService.next()`）。

### 2) 搜索聚焦快捷键

- [ ] 任意页面按 `Ctrl+F`，确认焦点切回曲库搜索框并高亮已有文本（如有）。
- [ ] 搜索框已聚焦时重复按 `Ctrl+F`，确认不会出现卡顿、焦点丢失或异常。

### 3) 冲突与可用性

- [ ] 文本输入控件聚焦时按播放快捷键，确认全局快捷键仍可触发且行为稳定。
- [ ] 连续快速按键（例如 `Ctrl+P` 连按 5 次），确认应用不崩溃、状态可恢复。
