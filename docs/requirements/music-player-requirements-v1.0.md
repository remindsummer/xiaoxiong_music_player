# 跨平台音乐播放器需求文档（V1.0）

## 1. 文档信息

- 产品名称：小熊音乐播放器
- 文档版本：V1.0（冻结版）
- 适用阶段：第一阶段（本地播放 MVP）
- 首发平台：Windows / macOS / Linux

## 2. 产品目标

本产品第一阶段面向本地音乐收藏用户，提供稳定、可持续使用的桌面端音乐播放体验。重点能力为本地曲库管理、高效检索、播放列表管理与歌词显示，兼顾后续联网能力扩展（当前仅用于歌词补全）。

## 3. 用户与场景

### 3.1 目标用户

- 以本地音乐收藏用户为核心
- 典型曲库规模约 2,000 首
- 对“管理效率 + 播放稳定性”敏感

### 3.2 核心使用场景

1. 用户导入本地目录后自动建库并可增量更新。
2. 用户按歌名/歌手/专辑快速检索并立即播放。
3. 用户维护多个播放列表并在不同场景下切换。
4. 用户在播放时查看同步歌词，本地缺失时可在线检索补全。
5. 用户通过应用内快捷键完成高频操作。

## 4. 信息架构与导航

第一版采用四主区信息架构：

- 曲库
- 播放列表
- 正在播放
- 设置

说明：

- 曲库承担浏览、检索、筛选与播放入口；
- 播放列表承担用户自定义集合管理；
- 正在播放承担当前歌曲详情、进度、歌词与播放控制；
- 设置承担主题、语言、目录、网络策略等参数配置。

## 5. 功能需求

### FR-1 本地曲库管理（必须）

1. 支持添加一个或多个本地音乐目录。
2. 支持手动全量扫描与增量更新。
3. 扫描结果写入本地索引并持久化。
4. 重启后可恢复曲库，无需重复扫描。
5. 最低元数据字段：文件路径、歌名、歌手、专辑、时长。
6. 封面字段为可选能力，不阻塞首版验收。

### FR-2 播放控制（必须）

1. 支持播放、暂停、停止、上一首、下一首。
2. 支持进度拖拽、音量调节、静音切换。
3. 支持顺序播放、单曲循环、随机播放三种模式。
4. 对损坏或不支持格式文件执行跳过策略，并给出可见提示。

### FR-3 播放列表（必须）

1. 支持创建、重命名、删除播放列表。
2. 支持添加/移除曲目。
3. 支持列表内排序调整。
4. 播放列表持久化并在重启后恢复。

### FR-4 搜索与筛选（必须）

1. 提供全局搜索入口。
2. 支持按歌名、歌手、专辑检索。
3. 搜索结果支持立即播放、加入播放列表。
4. 搜索结果与播放状态联动（当前播放项可识别）。

### FR-5 歌词系统（必须）

1. 采用混合策略：本地优先 + 在线补全。
2. 本地匹配规则：
   - 优先同名 `.lrc` 自动关联；
   - 支持用户手动指定歌词文件。
3. 在线策略：
   - 用户可发起在线检索；
   - 成功后缓存到本地并供后续复用。
4. 失败策略：
   - 在线检索失败后不自动重试；
   - 提供“手动重试检索”入口。
5. 播放中支持按时间同步滚动显示。

### FR-6 应用内快捷键（必须）

1. 第一版仅支持应用内快捷键，不支持系统全局热键。
2. 第一版采用固定预设键位，不支持用户自定义。
3. 必备快捷操作：
   - 播放/暂停
   - 上一首
   - 下一首
   - 聚焦搜索框
4. 设置页需提供快捷键说明清单。

### FR-7 设置中心（必须）

1. 基础设置项：
   - 主题
   - 语言
   - 默认扫描目录
   - 歌词联网开关
2. 所有设置本地持久化，并在下次启动生效。

## 6. 非功能需求

### 6.1 性能目标（2,000 首曲库，均衡档）

- 冷启动时间：<= 3 s（进入可交互主界面）
- 搜索首屏响应：<= 200 ms
- 切歌可听延迟：<= 300 ms

### 6.2 稳定性

- 播放异常文件时不得导致应用崩溃。
- 曲库加载失败时需保留可恢复路径（重试扫描或错误提示）。

### 6.3 可移植性

- 三端核心流程一致：导入、检索、播放、歌词、列表、设置。
- 允许少量平台差异（例如窗口控件样式），但不允许关键能力缺失。

### 6.4 隐私与网络

- 核心播放能力必须离线可用。
- 联网能力默认可控（可关闭）。
- 联网阶段仅用于歌词检索与缓存，不涉及账号与用户画像。

## 7. 范围边界（V1 不包含）

- 在线音乐播放
- 账号体系与云歌单同步
- 系统级全局快捷键
- 专业均衡器 / DSP 插件体系

## 8. 验收标准

满足以下全部条件即视为 V1 需求达成：

1. 可导入目录并稳定构建曲库，重启后曲库与播放列表可恢复。
2. 用户可从“搜索 -> 播放 -> 切换播放模式 -> 查看歌词”完成完整闭环。
3. 仅应用内快捷键可用，且至少覆盖播放/暂停、上一首、下一首、搜索聚焦。
4. 在线歌词失败时不会自动重试，且可手动触发重试。
5. 在关闭联网开关后，曲库与播放核心功能仍完整可用。
6. 在三端均可完成核心流程，且无阻断性崩溃。

## 9. 后续版本预留方向（非 V1 承诺）

- 快捷键自定义与冲突检测
- 歌词来源优先级策略细化
- 大曲库（10,000+）性能专项优化
- 全局媒体键支持
